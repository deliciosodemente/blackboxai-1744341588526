# Zapp - Interactive Video Mapping

## Project Overview
Zapp is an interactive video mapping application that allows users to transform spaces and create immersive experiences during events. This project utilizes modern web technologies to offer a visually stunning and engaging platform for video mapping.

## Installation
To set up the Zapp project locally, follow these steps:

1. **Clone the repository:**
   ```bash
   git clone <repository-url>
   ```
2. **Navigate to the project directory:**
   ```bash
   cd zapp
   ```
3. **Open the `index.html` or `index_v2.html` file in your web browser:**
   Ensure that your browser has access to the internet for loading external scripts.

## Usage
1. Open the `index.html` or `index_v2.html` file in your preferred web browser.
2. Explore the content and features available on the webpage.
3. Click on the "Learn More" and "Get Started" buttons to see alert messages providing additional information about the interactive video mapping technology and services.

## Features
- **Interactive Video Mapping:** Experience and explore interactive video mapping technology.
- **Responsive Design:** The application is designed with responsiveness in mind, adapting to different screen sizes.
- **User Engagement:** Includes buttons that interact with users and provide information through alerts.

## Dependencies
The project utilizes the following dependencies, as included in the HTML files:

- **Firebase:** For analytics and app functionality.
  - URL: `https://www.gstatic.com/firebasejs/9.6.1/firebase-app.js`
  - URL: `https://www.gstatic.com/firebasejs/9.6.1/firebase-analytics.js`

- **Tailwind CSS:** For styling the application.
  - URL: `https://cdn.tailwindcss.com`

## Project Structure
The project contains the following files:

```
/zapp
│
├── index.html           # First version of the interactive video mapping interface
├── index_v2.html       # Second version including Firebase configuration
```

### HTML File Breakdown
- **index.html**: Contains the basic structure, styling, and interactivity of the application without Firebase integration.
- **index_v2.html**: Enhanced version of the application featuring Firebase initialization and configurations that are necessary for further development.
